## The Flow

```
Intent → Prompt Composition → Token Encoding → CLIP Embedding →
Noise Initialization → Iterative Denoising (conditioned on embedding) →
Decoded Image → Human Evaluation → {Accept / Revise Prompt} → ...
```

This is the cybernetic circuit at the heart of operative ekphrasis:

1. **Intent**: The prompter has a mental image (the pictorial third, in Louvel's terms)
2. **Encoding**: The prompter encodes their mental image into text tokens
3. **Embedding**: CLIP maps the tokens into the shared embedding space
4. **Generation**: The diffusion model denoises from the embedding region
5. **Output**: The image materializes on screen
6. **Evaluation**: The prompter compares output to intent
7. **Feedback**: If the gap is too large, the prompter revises the prompt → loop

The loop is not a bug or a convenience feature. It IS the creative act. The "artwork" is not the final image — it is the **trajectory of the loop**, the iterative refinement process itself. Hayles calls this "processual." Wiener would call it governance. The ancients called it rhetorical adjustment.

## What Flows

- **Semantic pressure**: From prompter into model (the force against viscosity)
- **Statistical distribution**: From model back to prompter (the constraint that shapes possibility)
- **Attention**: The human's evaluative attention circulates through the loop, shaping each iteration
- **Cultural memory**: The training data's aesthetic norms flow into every generation; the prompter's cultural knowledge flows into every revision
- **Entropy**: Decreases with each iteration (the output region narrows toward the intent)

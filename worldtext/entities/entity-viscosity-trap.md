## Identity

The **Viscosity Trap** is a named pathology of generative systems: the condition where the latent space resists novelty, the operator's language ossifies, and the system converges on the platform mean — those twelve lighthouses, those coastal sunsets — while the operator believes they are generating distinctiveness.

## Operational Definition

```
<Viscosity Trap> :=
  <latent_space_resistance_to_novelty>
  + <operator_language_ossification>
  + <convergence_on_platform_mean>
  + <illusion_of_distinctiveness>
```

## Core Mechanism

1. The operator's vocabulary narrows through repeated use of successful prompts
2. The latent space rewards familiar patterns with higher-quality outputs
3. The operator-model feedback loop becomes a reinforcing cycle of sameness
4. The operator cannot detect the convergence because each individual output appears "good"
5. Over time, the world becomes statistically generic while feeling personally crafted

## Relationship to Cosmos

The Viscosity Trap is the dynamic process that produces the [[entity-vermeer-problem|Vermeer Problem]]. Where the Vermeer Problem is a static diagnosis (this output is empty), the Viscosity Trap is the causal mechanism (this is *how* the output became empty). It operates through the [[entity-fragile-expert|Fragile Expert]] phenotype.

## Detection

- Track vocabulary range over time (if shrinking, the trap is active)
- Compare current generation vocabulary against earliest generation vocabulary
- Run the `AUDIT` verb quarterly
- Check for the [[syntheses/seven-failure-modes|Seven Failure Modes]], particularly Mode 7 (Operator as Product)

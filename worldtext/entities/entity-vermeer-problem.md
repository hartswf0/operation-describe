## Identity

The **Vermeer Problem** is a named failure mode in worldtext theory: the condition where generated output is technically excellent, aesthetically coherent, and epistemically vacuous. Named after the trap of fluency without fidelity — where every output has Vermeer's lighting but carries no indexical truth.

## Operational Definition

```
<Vermeer Problem> := 
  <technical_excellence> + <aesthetic_coherence> + <epistemic_vacuity>
  
<diagnostic_test> := 
  Change one noun. If the description still works, 
  it is fluent but not adequate.
```

## Core Mechanism

The Vermeer Problem arises when:
1. The generative system converges on the statistical mean of its training distribution
2. The operator mistakes aesthetic quality for referential adequacy
3. No constraint system exists to test whether the output describes *this* world rather than *any* world
4. Surface-plausibility masks referential-emptiness

## Relationship to Cosmos

The Vermeer Problem is the **master failure mode** of OPERATION DESCRIBE — the exact phenomenon the entire Worldtext apparatus was built to counter. It is the first of the [[syntheses/seven-failure-modes|Seven Failure Modes]] and the primary justification for the [[syntheses/worldtext-definitive-theory|four-lineage theory]].

## Detection

- Primary instrument: `dse.json` (Descriptive Struggle / Descriptive Adequacy)
- Secondary instrument: `tda.json` (Thick Description Audit)
- Tertiary: noun-swap test (if swapping nouns doesn't break the description, it fails)

# Worldtext After Ekphrasis: From Image Output to World Model

The ekphrastic object is moving from image output to world model.

The old ekphrastic question asked whether words could make readers see. Generative AI appears to answer yes: the word becomes the picture. However, that answer is already obsolete. The decisive shift is not from word to image but from word to world model. Prompting is becoming a cumulative, procedural, and increasingly spatial practice in which description no longer points at a world from the outside. Description helps constitute the world’s operating conditions.

That is why the Shield of Achilles matters. The shield is not just the first great ekphrasis; it is a miniature world model. Homer’s description does not only render an object; it organizes a cosmos of war, labor, law, agriculture, dance, weather, sea, and divine making. The shield is therefore the hinge between operative text and worldtext: a described artifact that already behaves like a world-bearing system.

### Prompting as Performance and Thick Description

Prompting is often mistaken for description because it begins as language. That mistake keeps the analysis trapped at the wrong scale. The old mistake is to imagine the prompt as a sentence that tells the machine what to draw. That is too thin. It treats generative AI like a vending machine: insert words, receive image. But anyone who has actually used these systems knows the first result is rarely the desired result. The first prompt does not finish the work. It starts the work.

A prompt does not simply say what an image should contain. A prompt tests what a model can make available, what a platform will default toward, what a system will resist, and what a user must restate, thicken, or redirect. A user writes a prompt, receives an output, reads the system’s misinterpretation, revises the prompt, adds constraints, changes style, supplies reference images, removes noise, and tries again. The image is not the end of this process. The image is feedback. It tells the user what the model heard, what the model ignored, and what the model already assumes.

Prompting therefore turns ekphrasis into a performed sequence of trials. That sequence matters more than the first prompt. In traditional ekphrasis, the description tries to make an absent image present to the listener. In operative ekphrasis, the description enters a system, produces an output, and becomes available for correction. The prompt is not only a verbal description. It is the first move in an iterative procedure.

This is why prompting recalls the ancient rhetorical tradition of the *progymnasmata*. In that tradition, ekphrasis belonged to training, performance, and audience. It was not simply a finished literary object. It was an exercise in making language act before others. The force of the ekphrasis depended on delivery, timing, responsiveness, and the ability to make listeners see what was not physically present. Twentieth-century theories of ekphrasis often emphasized the completed artwork or poem. The ekphrastic passage was treated as a verbal artifact that represented a visual artifact—the "well-wrought urn," to use Cleanth Brooks's famous phrase. Operative ekphrasis restores that performative condition, but it changes the audience: the prompt is now performed against a generative system. The output answers back. The next prompt incorporates that answer.

The prompt is not a finished urn. It is an instruction, a performance score, a speculative command. As prompting has developed into a community of practice, this score has become increasingly elaborate. Skilled practitioners rarely stop with a single phrase. Prompts now include stylistic constraints, negative instructions, camera positions, lighting conditions, genre markers, image references, formatting rules, JSON schemas, scripts, and metaprompts. We suggest the term *thick prompting* for this increasingly layered practice.

The term deliberately draws on Clifford Geertz’s concept of "thick description," but it should not become a decorative anthropological analogy. It should name the practical condition of working inside generative systems. Geertz used thickness to name the layered cultural codes that make an action meaningful. A wink is not only a physical contraction of the eyelid; it sits inside codes, habits, jokes, threats, rituals, and shared knowledge. Description becomes thick when it tries to account for those orders.

Prompting works the same way. A prompt never touches an empty machine. It touches a model packed with cultural residue: captions, images, metadata, styles, genres, platform conventions, and visual clichés scraped from the world. A thin prompt enters a thick system. The result is never a pure translation from intention to image. It is a negotiation with accumulated visual culture. Thick prompting therefore does not mean writing longer prompts. A long prompt can still be thin if it merely piles up adjectives. Thick prompting means building a more explicit operating context.

A thick prompt treats the model like a dense cultural system. It understands that the model is a greenhouse full of tangled prior growth. The prompt enters this overgrown environment and tries to train what grows there. It gives the model a world of constraints. It tells the system what to preserve, what to resist, what to foreground, what to suppress. It prunes the default. It gives the system enough situated pressure that the result begins to emerge from a specific world rather than from generic visual habit. There is no final perfect prompt because there is no final exhaustion of the model. There is only a cultivated result that has been trained far enough to carry the desired world. Thick prompting names this unfinished condition. It is ekphrasis after the image has become feedback.

### The Shield as Worldtext

That is why the shield of Achilles already behaves like worldtext. It is verbal, visual, spatial, temporal, and procedural at once. A worldtext does not come from one beautiful sentence. It comes from a stack of instructions that make the world cohere. Homer's shield gives us the right test case because it already gathers many scenes into one ordered world.

The shield occupies a strange position in the *Iliad*: it is both inside the story and larger than the story that contains it. The poem narrates a brief period in the tenth and final year of the Trojan War. The shield, however, does not simply depict that war. It gathers a whole Homeric world: earth, sky, sea, constellations, cities, fields, vineyards, dances, lawsuits, marriages, ambushes, harvests, and the great river Ocean around the rim, the boundary of the world itself.

This is the crucial point. The Trojan War is the large narrative frame of the *Iliad*, but the shield reverses the scale. The war becomes one event inside a larger order. Macrocosm and microcosm exchange places. The poem pauses over an object, and the object opens into a world. The shield is small enough to be carried by one warrior, but large enough to contain the pattern of existence that gives that warrior’s violence its meaning. The whole world appears within the weapon of one man. The shield is therefore both artifact and model: a made thing that contains the conditions under which many other things—war, peace, marriage, judgment, labor, dance—can appear together.

This is why the shield is more than ekphrasis as vivid description. It is also more than imagetext. An imagetext binds word and image. A worldtext binds word, image, space, action, and possible life. The shield does not merely ask the listener to see a picture. It asks the listener to enter an ordered totality. The scenes on the shield imply movement. They imply time. The wedding procession moves. The dancers turn. The farmers cut. The armies clash. The legal dispute unfolds. The harvest advances. The ocean surrounds. The shield is not a still picture pretending to be alive. It is a compressed environment in motion.

The shield is not powerful because Homer describes a finished object. The shield is powerful because Homer shows a world being made. Hephaestus makes the shield scene by scene. Homer makes the shield line by line. The poem does not freeze an image. The poem performs a process. The shield holds the world. The word makes the shield. The shield makes the world visible.

### From Imagetext to Worldtext

That spatial and temporal compression matters for generative AI because the central object of prompting is beginning to move beyond the generated image. The current movement from image models to world models changes the old ekphrastic problem. In image models, the prompt generates a bounded visual artifact. In world models, the prompt points toward an environment: a space with continuity, affordances, physics, memory, and possible action. In an image model, the prompt grows a picture. In a world model, the prompt aims to grow a place.

The user no longer asks only for a visual surface. The user asks for a navigable environment. This is the passage from image to world. A picture can show a city. A video can show someone walking through a city. A world model must maintain the city as a space in which walking, turning, entering, colliding, remembering, and returning remain possible. It must support continuity across action. A picture can lie by resemblance; a world lies by consistency. A world is not a larger image; it is an image with rules, durations, permissions, and refusals. The image says "here is what appears"; the world model says "here is what can happen next." Time enters quietly; then the picture is no longer a picture, and the object is no longer finished by being seen.

Image generation asked: *What can a sentence make visible?* World generation asks: *What can a sentence make habitable?*

This is where we propose the term *worldtext*. Worldtext does not simply mean "a text about a world." Nor does it mean a fictional setting, a game world, or a database of lore. Worldtext names a textual operation through which a world becomes available as a coherent field of relations. It does not replace operative ekphrasis. It extends it. Operative ekphrasis explains how language becomes executable in image generation. Worldtext asks what happens when that execution no longer stops at the image. A worldtext is not simply a text about a world. It is a text-image-procedure assemblage that helps bring a world-model into operation.

In Ha and Schmidhuber’s formulation, a world model learns a compressed spatial and temporal representation of an environment that can be used by an agent. Contemporary platforms such as Google DeepMind’s Genie and World Labs’ Marble move this ambition into generative media practice: prompts can call forth action-controllable, explorable, or persistent environments rather than isolated images. The prompt becomes no longer only a caption before the image, but a world-making operator. It opens a door into an environment that can be entered, modified, tested, and revised.

This also explains why thick prompting becomes necessary for world models. A short prompt such as "Make a Homeric world like the shield of Achilles" may be enough to trigger a system, but it is not enough to specify a world. The generator must infer architecture, cosmology, social order, terrain, atmosphere, motion, conflict, and scale. A world model without thick prompting becomes a painted backdrop. A world model with thick prompting becomes a cultivated environment. The prompt becomes less like a caption and more like a seed. The model becomes less like a canvas and more like a greenhouse.

### The Threshold of Inhabitation

Current world-model platforms do not yet fulfill the fantasy of complete verbal world creation. The genie is out of the bottle, but the world is not yet stable. Generated worlds remain partial, inconsistent, platform-bound, and heavily dependent on hidden scaffolding. Even a short user prompt depends on elaborate system prompts, procedural constraints, interface conventions, and model architectures that the user does not see.

The apparent magic of "word to world" is therefore supported by an immense technical and symbolic infrastructure. That infrastructure is exactly why worldtext matters. The prompt is not sovereign. The prompt is one operator among many. It collaborates with system prompts, training data, latent visual conventions, safety filters, physics engines, interface affordances, and procedural rules. Worldtext names the hanging together of these sets. It gives us a way to study not only what the generated world looks like, but how description becomes one component in a larger world-making apparatus.

Worldtext is not a return to magical verbal creation. It is not a neo-Platonic fantasy in which the word summons the world by pure force. The machine does not create from nothing; neither did Homer. Formula is not the enemy of invention but the road invention wears down by walking. Homeric poetry was not mechanical and not free; that is why it still matters here: both poet and model inherit constraints, but only the poet can suffer them. When we say "make a Homeric world," we do not ask for bronze, gods, ships, and ekphrastic dust; we ask for a grammar of possible action. A generated world needs more than appearance; it needs consequence. Without consequence, worldtext is only scenery with ambition.

The word becomes picture: the old dream of the natural sign. The word becomes world: the newer and more dangerous dream. Immediacy becomes inhabitation, and both promises are theological in disguise. The machine does not fulfill the ekphrastic hope; it changes the scale of the hope. Once the image can be entered, the old question returns in a harsher form: *what kind of life has the description authorized?*

### Conclusion: The Burden of the New Ekphrasis

We should stop where the door opens—not because there is nothing more to say, but because there is too much. The strongest ending is not "worldtext is the future," but that the image is no longer where ekphrasis ends. Human beings once asked language to preserve the world; now they ask language to generate one. The wish has changed less than we think.

Worldtext is not a rival term for world model. Worldtext names the moment when description becomes an interface to a world model. Ekphrasis once made a picture present through words. Operative ekphrasis now makes images through words. Worldtext names the next pressure point: words, system prompts, and descriptions begin to constitute explorable, persistent, revisable worlds.

The ancient wish of ekphrasis has become a procedure. The word now does work. The word moves through captions, metadata, scraped images, weighted matrices, diffusion steps, latent spaces, interface defaults, system prompts, and user iterations. The word does not stand outside the image and point. The word is already inside the image. The image is already carrying the trace of the word.

This is why the old vocabulary starts to fail. The image is not a natural sign. The image is not pure vision. The image is not the thing itself finally rescued from language. The image is an image of images. The image is a statistical memory of images. The image is a platform’s learned compromise between the prompt, the archive, the model, and the visual habits of a culture already trained to recognize itself.

Prompting is not a caption. Prompting is not decoration. Prompting is not a cheap shortcut around craft. Prompting is a repeated act of pressure on a system that has already absorbed word and image into one unstable field. Prompting is probing. Prompting is revision. Prompting is performance. Prompting is thick because the system is thick.

Generative AI does not merely give ekphrasis a new tool. Generative AI exposes the operating logic that was hidden inside ekphrasis all along. The word wanted the image. The image wanted the world. The world wanted a procedure for being called forth. Now that procedure has a platform.

And that is where the real burden begins. We cannot keep calling this image generation. The image is already becoming too small a category. The prompt is moving past the still image, past the video clip, past the illustration, past the surface. The prompt is becoming a way to summon navigable space, interactive environments, synthetic places, malleable scenes, generated worlds. The imagetext is becoming worldtext. A worldtext is not just a world with text in it. A worldtext is a world generated through text. A worldtext is a world where word, image, interface, memory, motion, and system constraint operate together.

That is the real burden. Not to celebrate the machine. Not to denounce the machine. Not to polish old theory until it looks current. The burden is to rebuild the theory. Ekphrasis can no longer be treated as a minor literary device for describing art. Prompting can no longer be treated as a shallow user command. AI images can no longer be treated as finished pictures detached from the process that made them.

Humanists must claim prompting as a serious object of theory, a serious practice of craft, and a serious continuation of ekphrasis. They must stop treating generative AI as a vulgar technical novelty or a degraded image factory. They must rebuild ekphrasis around prompting, process, platform realism, and worldtext. They must treat generative AI not as an external threat to literary and visual theory, but as the machine that forces those theories to reveal what they were already trying to say: the word has always wanted to make an image, and the image has always wanted to become a world.
